
# 产品需求文档 (PRD)：WildWeChat MD（狂野排版机）- MVP 版

### 1. 产品概述

* **产品名称：** WildWeChat MD（暂定名，反正是你自己用）
* **一句话介绍：** 一个纯前端、零依赖云端、专为微信公众号深度定制的 Markdown 样式注入与剪贴板劫持工具。
* **解决的痛点：** 现有的排版工具（如 mdnice、doocs）样式封装过重，无法满足 PM 对极客化、狂野 CSS 风格的绝对控制欲，且操作链路包含多余的“云端上传”动作。
* **核心产品理念：** Serverless（无后端）、Zero-UI（极简界面）、CSS in Control（样式绝对接管）。

### 2. 用户与使用场景

* **目标用户：** 仅限创作者本人（懂基础 HTML/CSS，熟悉 Markdown 语法的极客 PM）。
* **核心场景：** 用户在本地（如 Obsidian / VS Code）完成 Markdown 文章撰写及公网图床的图片插入。打开本工具，将纯文本一键转换为带有强自定义 CSS 的内联富文本，并直接粘贴至微信公众号后台发布。

### 3. 用户操作流程 (User Flow)

这是一个线性的、不可逆的操作流，不需要任何状态保存：

1. **输入：** 粘贴纯 Markdown 文本至左侧输入区。
2. **渲染与注入（静默发生）：** 系统实时将 MD 解析为 HTML，并读取内置的 `custom.css`，将样式强行转化为 Inline Style 注入 HTML 标签。
3. **预览：** 右侧实时呈现公众号最终的视觉效果。
4. **输出：** 点击【一键复制】。
5. **终点：** 切换至微信公众号网页后台，`Ctrl+V` 粘贴。

### 4. MVP 功能需求列表（Feature List）

| 模块 | 功能项 | 详细说明与验收标准 | 优先级 |
| --- | --- | --- | --- |
| **UI 层** | 双栏极简面板 | 页面分左右两栏，左侧为 `<textarea>`，右侧为 `<div id="preview">` 渲染区。顶部悬浮一个【一键复制微信排版】按钮。 | P0 |
| **解析层** | MD 转 HTML | 引入 `marked.js`。**验收标准：** 必须完美支持 GFM 语法（特别是表格和代码块），且不能过滤掉原文中的 HTML 注释。 | P0 |
| **样式层** | **内联样式注入器** | 引入 `juice` 库。**验收标准：** 读取本地预设的 CSS 文本，遍历生成的 HTML 树，将类名和标签样式100%转化为 `style="..."` 属性。 | P0 |
| **输出层** | 剪贴板劫持 | **验收标准：** 点击复制按钮后，必须使用 `Clipboard API` 写入 `text/html` MIME 类型的数据包。如果写入的是纯文本，则视为 Bug。 | P0 |
| **拓展层** | CSS 热更新（可选） | 左侧增加一个 Tab，允许直接在网页里修改 CSS 代码并实时看右侧效果。（如果不嫌麻烦，这步可以不做，直接在本地改文件）。 | P1 |

### 5. 绝对边界与约束（Not To Do List）

作为 MVP，以下功能**绝对不允许**出现在第一版代码中，否则项目必定延期且脱离初衷：

1. **不做图床代理：** 不处理任何本地图片的上传逻辑。如果图片裂了，是用户（你自己）在写 MD 时的路径问题。
2. **不做账号体系与云端存储：** 文本和 CSS 全部跑在浏览器内存里，刷新即重置。
3. **不支持伪元素与外部字体：** CSS 中严禁使用 `::before`、`::after`、`@import url()`，因为微信后台的解析器会直接把这些属性物理阉割。
